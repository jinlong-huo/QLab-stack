# Figure 规范

组内论文图的统一标准。目标是：**关掉正文，只看图就能大致理解这篇论文。**

---

## 设计原则

1. **自包含** — figure + caption 构成独立叙事。不看正文也能懂。
2. **最小化** — 每张图只讲一件事。不要在一张图里塞 3 个信息。
3. **可复现** — 所有图的生成代码放在对应 repro/ 目录下。

---

## 线图 (Line Plot)

```
✅ 正确做法:
- 坐标轴标签清晰（不是变量名，是含义: "Latency (ms)" 不是 "lat"）
- 图例在边框内第一位不挡数据
- 多条线用线型+颜色双重区分（色盲友好）
- 误差线可见
- 字号和正文相当（不是默认 8pt 小字）

❌ 常见错误:
- 图例把数据盖住了
- 只有颜色没有线型区分（打印成黑白就废了）
- y 轴不从 0 开始但不做标注（误导）
- "see text for details" — 如果需要在 caption 里解释，说明图本身不够清楚
```

---

## 架构图 (System/Architecture Diagram)

```
画架构图之前先回答三个问题:
1. 数据流从哪到哪？（箭头方向一致，不交叉）
2. 哪些是已有组件，哪些是我们新加的？（新组件用实线/深色，旧组件用虚线/浅色）
3. 读者第一眼应该看哪里？（用颜色/粗细引导视线）

工具: draw.io / Figma / TikZ（出版级）
不要用 PPT 画图 — 导出 PDF 时字体会乱。
```

---

## 表格

```
✅ 正确做法:
- 最优结果加粗
- 第二优加下划线
- 表头清晰，有单位
- 数值精度合理（93.2% 不是 93.187654321%）

❌ 常见错误:
- 全是文字没有数字的比较表 — 这种改写成段落
- 表太大横跨两页
```

## 颜色

统一调色板（色盲友好，打印安全）：

| 用途 | 色号 | 颜色 |
|---|---|---|
| 主色 | `#1f77b4` | 蓝 |
| 辅色 | `#ff7f0e` | 橙 |
| 第三色 | `#2ca02c` | 绿 |
| 对比色 | `#d62728` | 红 |
| 背景 | `#f0f0f0` | 浅灰 |

> 这是 Tableau 10 调色板的前 5 色。Matplotlib 默认就是这个。不要自己发明颜色。

## Checklist（投稿前逐张图过）

- [ ] 打印成黑白后还能区分每条线/每个颜色吗
- [ ] Caption 独立可读吗（不依赖正文）
- [ ] 文字大小和正文一致吗
- [ ] 生成这张图的脚本在 repro/ 目录下吗
- [ ] 没有"见图 X"但图 X 在 3 页之后
