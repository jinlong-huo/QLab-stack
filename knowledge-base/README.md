# Knowledge Base

组内知识沉淀。不按论文排列，按**主题**组织——一个主题下汇总所有相关论文的关键结论。

## 目录

- `topics/` — 按研究方向分类的主题文档
- `glossary.md` — 术语表
- `reading-roadmap.md` — 新人阅读路线图

## 主题文档格式

每个 `topics/xxx.md` 包含：
1. **定义** — 这个问题是什么
2. **关键论文时间线** — 按年份排列的里程碑论文 + 一句话贡献
3. **当前 SOTA** — 现在最好的方法是什么
4. **开放问题** — 还有什么没解决
5. **组内相关工作** — 我们组在这个方向上的论文/项目

## 贡献方式

- 读完一个方向的 5+ 篇论文后，写一篇主题文档
- 在 reading-roadmap 里加上这个方向的推荐阅读顺序
- 遇到新术语及时更新 glossary
