# How to Choose a Problem

> 选问题可能是 PhD 期间最重要的技能。选对了，事半功倍。选错了，白干半年。

---

## 好问题的四个标准

一个好问题应该同时满足：

1. **重要的** — 即使你不是做这个的，你也会承认这个问题值得解决
2. **未被解决的** — 不是"调参能把 SOTA 刷上去"的问题，是"没人知道怎么做"的问题
3. **你有独特视角的** — 你比别人多知道一点什么（工具、场景、数据、insight）
4. **可验证的** — 你能在 3-6 个月内做出可展示的结果

## 问题来源（按可靠性排序）

1. **读论文时的 "Wait, that can't be right"** — 论文声称的结论和你的直觉冲突。追下去。
2. **做实验时的 "This should work but doesn't"** — 系统表现和理论预测不符。追下去。
3. **导师建议** — 但你需要自己判断是否符合上面的四个标准
4. **顶会 "Future Work" 章节** — 通常已经被那个组自己做了，但偶尔有漏网之鱼

## 红线（碰到就跑）

- "还没有人用 X 方法做 Y 任务" — 可能没人做是因为做不了。先花 1 周做 feasibility check。
- 问题定义太模糊 — 如果不能用一句话说清输入和输出，问题还没定义好。
- 需要 10 个 GPU 跑 baseline — 先确认资源再定方向。

## 对齐导师

在真正投入之前，写一段话给导师：

```
问题: [一句话]
现有方法为什么不行: [一句话]
我的核心 idea: [一句话]
最简单的验证实验: [一个能在 2 周内做完的实验]
失败条件: [如果 X 结果不对就放弃这个方向]
```

导师回 OK 再开始。不要闷头做 3 个月再拿出来。
