# Welcome

欢迎加入 [组名]。这篇文档告诉你我们怎么工作。

## 组内文化

- **写下来** — 讨论结论、设计决策、论文笔记，都写成文档。口头讲完不写 = 没发生过。
- **先读再问** — 提问之前先查 glossary 和 knowledge-base。如果找不到答案，说明我们文档有缺口——修好它，这样下一个人的问题就少一个。
- **模板驱动** — 不要从空白页开始。论文笔记有模板，周报有模板，会议记录有模板。填空就行。
- **迭代优于完美** — 一个不完美的 PR 比一个完美的想法有价值。先发出来，再改进。

## 节奏

| 事项 | 频率 | 说明 |
|---|---|---|
| 周报 | 每周五 | 用 templates/weekly-report.md 格式 |
| 组会 | 每周一 | 轮流讲论文，用 templates/paper-presentation.md |
| 论文笔记 | 每读完一篇 | paper-notes/ 下提交 |
| 知识库更新 | 随时 | glossary 缺术语就加，reading-roadmap 有推荐就补 |

## 成长路径

```
新生 (0-3月)    → 能独立读论文 + 写笔记 + 复现实验
成员 (3-12月)   → 能提出自己的 research idea + 带新生读论文
核心 (12月+)    → 能定方向 + 写 proposal + 对外输出（论文/开源）
```
