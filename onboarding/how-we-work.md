# How We Work

## 沟通

- **异步优先** — 能用 issue/PR comment 解决的不开会
- **决策写下来** — 任何技术决策（选方案 A 不选 B）在对应 issue 里写清楚理由
- **公开频道** — 技术讨论放在群聊/issue 里，不要私聊。让其他人能搜到

## 代码

- **每个 repo 有 README** — 新成员 3 分钟能跑起来
- **每个 PR 有描述** — 做了什么、为什么、怎么测
- **review 24h 内完成** — 别让 PR 烂在队列里

## 文档

- **模板驱动** — 论文笔记、周报、会议记录都用模板
- **写完就发 PR** — 不追求完美，有人 review 比一个空着的 "TODO" 强
- **知识库持续更新** — glossary 缺术语就加，reading-roadmap 有推荐就补

## 会议

| 会议 | 频率 | 时长 | 产出 |
|---|---|---|---|
| 组会 | 每周一 | 1h | 1 人讲论文，全员讨论，记录在 meeting-notes |
| 1-on-1 | 每周 | 30min | 和导师单独对齐 |
| 技术讨论 | 按需 | ≤1h | 议题提前写在 issue 里，结论写在 issue 里 |

## 论文

- **每天扫 arXiv digest 邮件** — 10 秒扫标题，感兴趣的存 Zotero
- **每周至少精读一篇** — 写 paper-notes，提交 PR
- **组会讲 paper** — 用 paper-presentation 模板，15min 讲 + 15min 讨论
