# Exit Checklist

> 离组前逐项确认。和导师一起过一遍。

## 代码 & 项目

- [ ] 每个活跃 repo 指定新 maintainer
- [ ] README 写清楚怎么跑（环境、依赖、入口脚本）
- [ ] 正在跑的实验 → 写清楚当前状态、剩余步骤、已知坑
- [ ] 更新 [projects.md](../projects.md)：项目改 owner 或标为"已结束"
- [ ] 私人项目细节从 `members/<name>/projects/` 交给接手人或归档

## 论文笔记

- [ ] 已读但没写笔记的论文 → 至少补一句摘要（放 `members/<name>/paper-notes/backlog.md`）
- [ ] 确认 `members/<name>/paper-notes/` 目录完整

## 知识交接

- [ ] 填完 [knowledge-handover.md](knowledge-handover.md)
- [ ] 和接手人 pair：让对方成功跑通一个实验 / 复现一个结果
- [ ] 补 glossary：你熟悉的术语，glossary 里缺的，加上
- [ ] 补 reading-roadmap：你觉得值得推荐但 roadmap 里没有的论文，加上

## 权限清理

- [ ] 共享账号、API key、服务器权限 → 确认回收
- [ ] GitHub / GitLab 权限 → 确认调整
- [ ] 邮件列表 / Slack / 微信群 → 确认退出

## 提取个人内容

- [ ] 运行 [extract.sh](extract.sh) 导出 `members/<your-name>/` 到本地
- [ ] 确认导出内容完整

## 退出访谈

- [ ] 和导师一对一
- [ ] 反馈组内流程哪里可以改进（可选）
